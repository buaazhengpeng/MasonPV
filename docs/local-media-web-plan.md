# 个人本地图片与视频网页开发策划

## 1. 项目定位

本项目用于制作一个面向个人使用的本地图片与视频浏览网页。核心目标是把本机或家庭服务器上的媒体文件以网页形式组织、浏览、筛选和播放，优先保证本地优先、隐私安全、结构清晰和后续可持续扩展。

### 1.1 使用场景

- 在浏览器中查看本地照片、截图、相册、短视频和长视频。
- 按目录、时间、类型、标签、收藏状态快速筛选媒体。
- 使用缩略图网格快速浏览图片和视频封面。
- 点击媒体进入详情页，查看大图、播放视频、查看文件基础信息。
- 后续可扩展为家庭局域网媒体站或个人 NAS 前端。

### 1.2 项目边界

首个版本聚焦本地媒体索引与浏览，不包含复杂社交、云同步、多人权限体系和在线编辑能力。文件本身保留在本地路径，网页系统只读取索引、生成缩略图和提供浏览入口。

## 2. 产品功能规划

### 2.1 MVP 功能

1. 媒体目录配置
   - 支持配置一个或多个本地媒体根目录。
   - 支持图片与视频文件类型白名单。
   - 支持忽略隐藏目录、缓存目录和指定排除路径。

2. 媒体扫描与索引
   - 扫描本地文件并生成媒体索引。
   - 记录文件路径、文件名、类型、大小、修改时间、创建时间。
   - 支持增量扫描，避免每次全量重建。

3. 缩略图与预览
   - 图片生成缩略图。
   - 视频截取封面帧。
   - 缩略图缓存在本地应用目录。

4. 媒体浏览页面
   - 首页展示最近媒体。
   - 支持图片/视频类型切换。
   - 支持目录导航。
   - 支持网格视图和基础分页/无限滚动。

5. 媒体详情页面
   - 图片大图预览。
   - 视频在线播放。
   - 展示文件名、路径、大小、分辨率、时间等元信息。
   - 支持打开原始文件所在目录的后续扩展接口。

6. 搜索与筛选
   - 按文件名搜索。
   - 按媒体类型筛选。
   - 按时间倒序排序。

7. 本地隐私与安全
   - 默认仅监听 `localhost`。
   - 不上传原始媒体文件。
   - 配置文件不提交真实个人路径。

### 2.2 后续增强功能

- 收藏、标签、评分。
- 相册集合与智能分组。
- EXIF 信息读取与拍摄时间排序。
- 视频时长、编码、分辨率展示。
- 重复文件检测。
- 人脸识别、地点聚合等本地 AI 能力。
- PWA 离线访问与移动端适配优化。
- 局域网访问模式与访问密码。

## 3. 技术栈策划

### 3.1 前端

推荐技术栈：

- 框架：Next.js + React
- 语言：TypeScript
- 样式：Tailwind CSS
- UI 组件：shadcn/ui 或 Radix UI
- 状态管理：React Server Components + URL Query + 少量 Zustand
- 媒体展示：
  - 图片：原生 `img` / Next Image 按实际文件访问方式评估
  - 视频：原生 `video`
  - 虚拟列表：TanStack Virtual
- 数据请求：TanStack Query 或 Next.js Server Actions

选择理由：

- Next.js 适合同时处理网页和本地 API。
- TypeScript 便于维护媒体元数据模型。
- Tailwind CSS 可快速搭建响应式网格页面。
- 后续可以平滑扩展桌面壳或局域网部署。

### 3.2 后端与本地服务

推荐技术栈：

- 运行时：Node.js
- API：Next.js Route Handlers
- 本地文件扫描：Node `fs/promises`
- 媒体元信息：
  - 图片尺寸：sharp
  - 视频元信息与截图：ffmpeg / ffprobe
- 缩略图生成：sharp + ffmpeg
- 任务调度：轻量队列，首版可用数据库任务表或内存队列

选择理由：

- 前后端同仓库，降低首版复杂度。
- Node 对本地文件系统访问直接。
- sharp 和 ffmpeg 是成熟的媒体处理工具。

### 3.3 数据存储

推荐技术栈：

- 数据库：SQLite
- ORM：Prisma 或 Drizzle ORM
- 缓存目录：应用本地 `.cache` 或用户配置目录

核心数据表建议：

- `media_items`
  - `id`
  - `root_id`
  - `relative_path`
  - `absolute_path_hash`
  - `filename`
  - `media_type`
  - `mime_type`
  - `size`
  - `width`
  - `height`
  - `duration`
  - `mtime`
  - `ctime`
  - `thumbnail_path`
  - `created_at`
  - `updated_at`

- `media_roots`
  - `id`
  - `name`
  - `path`
  - `enabled`
  - `created_at`
  - `updated_at`

- `scan_jobs`
  - `id`
  - `status`
  - `started_at`
  - `finished_at`
  - `total_files`
  - `processed_files`
  - `error_message`

- `tags` 与 `media_tags`
  - 用于后续标签扩展。

### 3.4 配置方式

建议使用本地配置文件：

```json
{
  "mediaRoots": [
    {
      "name": "Photos",
      "path": "/path/to/photos"
    },
    {
      "name": "Videos",
      "path": "/path/to/videos"
    }
  ],
  "includeExtensions": [".jpg", ".jpeg", ".png", ".webp", ".gif", ".mp4", ".mov", ".mkv", ".webm"],
  "excludePatterns": ["**/.DS_Store", "**/node_modules/**", "**/.cache/**"],
  "listenHost": "127.0.0.1",
  "listenPort": 3000
}
```

真实路径建议只保存在本地未跟踪配置中，例如 `config.local.json`，仓库只提交 `config.example.json`。

4. 配置路径别名，例如 `@/components`、`@/lib`。
5. 添加 `.env.example`、`config.example.json`、`.gitignore`。

### 5.2 数据层

1. 引入 SQLite。
2. 选择并配置 ORM。
3. 定义 `media_roots`、`media_items`、`scan_jobs` 数据模型。
4. 编写数据库初始化脚本。
5. 添加基础数据访问函数。

### 5.3 媒体处理层

1. 编写配置读取模块。
2. 编写文件扫描器。
3. 添加扩展名过滤和排除规则。
4. 实现图片元信息读取。
5. 实现视频元信息读取。
6. 实现缩略图生成与缓存。

### 5.4 API 层

1. `GET /api/media`：媒体列表。
2. `GET /api/media/:id`：媒体详情。
3. `GET /api/media/:id/file`：读取媒体文件流。
4. `GET /api/media/:id/thumbnail`：读取缩略图。
5. `POST /api/scan`：触发扫描。
6. `GET /api/scan`：查看扫描状态。

### 5.5 页面层

1. 首页媒体网格。
2. 顶部筛选栏。
3. 目录导航。
4. 媒体详情页。
5. 设置页展示媒体目录配置和扫描入口。

## 8. 风险与注意事项

### 8.1 本地路径安全

- 不要把真实个人目录写入 Git。
- API 返回路径时优先返回相对路径或脱敏路径。
- 文件读取接口必须校验目标文件位于已配置媒体根目录下，避免任意文件读取。

### 8.2 性能风险

- 大量媒体扫描可能耗时，应记录进度。
- 缩略图生成应缓存，避免重复处理。
- 前端列表需要分页或虚拟滚动。
- 视频文件不应一次性读入内存，应使用流式响应。

### 8.3 依赖风险

- ffmpeg 可能需要系统安装。
- sharp 在不同平台可能存在原生依赖差异。
- 后续 agent 搭建环境时应检查 Linux、macOS、Windows 的兼容性。

### 8.4 隐私风险

- 默认只监听本机地址。
- 若开放局域网访问，应增加访问密码。
- 不应默认生成公开分享链接。

## 10. 推荐实施顺序

先完成可运行的最小闭环：配置目录 -> 扫描文件 -> 写入数据库 -> 列表展示 -> 点击预览。随后再扩展缩略图质量、筛选体验、标签收藏和移动端适配。这样可以尽早验证本地文件读取、浏览器播放和缩略图缓存这些核心风险点。
